#アプリ名：ファミリーステップ

##サービス概要

このサービスは、加齢による健康への不安を抱える人とその家族のためのアプリです。
家族同士がつながり、お互いに励まし合うことで、運動習慣を身につけるきっかけをつくります。
大切な家族が健康に過ごせる未来を支えるサービスを目指しています。

##開発背景&解決したい課題

理学療法士として勤務していた際、病気によって自宅に帰りたいという希望があっても、介護負担などの理由から自宅に戻れない患者さんを多く見てきました。
その状況を前に、もっと早い段階から健康に気をつけることができれば防げたのではないかと感じる場面があり、何もしてあげられない無力感を覚えました。

病気には様々ありますが、生活や人生を大きく変えてしまう可能性があります。また、病気は本人だけでなく家族にも介護や通院、医療費など大きな負担をもたらします。

しかし、多くの人は「まだ健康だから大丈夫」と考え、将来起こり得る病気に対して日常的な予防行動を取る機会が少ないのが現状です。

そこで本サービスでは、家族同士がつながり、運動の記録やリアクションを通して互いに励まし合うことで、日常生活の中で自然に運動習慣を身につけるきっかけを作ることを目指しています。
運動を通じて健康寿命を延ばし、将来の病気や介護リスクを少しでも減らすことを目標としています。

##サービス詳細

親は散歩や簡単な運動を記録し、家族はその記録にリアクションを送ることで互いに励まし合うことができます。
散歩などの軽い運動を日常生活の中に取り入れることで、健康維持と生活習慣の改善を目指します。
また、体重や血圧の数値などを簡単に記録することで、自身の健康状態の変化を意識するきっかけを作ります。
家族が伴走する仕組みによって、無理なく運動を続けられる環境を提供します。

##ターゲット層

メインターゲット
・50〜70代の親世代と、その家族、30〜50代の子ども、または孫。
・離れて暮らしているが、お互いの健康を気にかけている家族。
・日常生活の中で散歩や軽い運動を取り入れ、健康維持をして欲しいと考える子ども。

ターゲット層の理由
・50〜70代は生活習慣病や身体機能の低下が現れやすく、運動習慣を身につけることで健康寿命の延伸が期待できる年代であるため。
・離れて暮らす家族は健康状態が見えにくく、互いに「健康でいてほしい」という思いが行動の動機になりやすいため。

##サービスの差別化ポイント・推しポイント
既存サービスとの比較

・既存の見守りアプリは、家族の安否確認や生活状況の共有を主な目的としている。
・歩数共有や位置情報などにより「異変があったときに気づく」仕組みが中心。
・健康管理よりも、安全確認やコミュニケーション機能に重点が置かれている。

本サービスの特徴

・家族で運動を見守り合い、日常の中で運動習慣を作ることを目的としている。
・歩行や軽い運動の記録を共有し、家族のリアクションによって継続のモチベーションを生み出す。
・血圧や体重のデータなどを記録し、自分の健康状態を意識することで将来の介護予防につなげる
「見守り」ではなく「介護を防ぐ」ための家族健康アプリです。
・運動の記録は、実施した運動項目のボタンを押すだけ、手間を取らせない設計としている。

##サービス利用の流れ（現在のイメージ）

ユーザー登録
家族グループ作成
親に招待リンクを送る
実施した運動のボタンを押す
リアクションを送り合う

##機能

###MVPリリース

ユーザー登録/ログイン
家族のグループ作成
運動項目のボタン入力（散歩、椅子スクワット、踵上げ）
運動データのグループ共有
家族間でのリアクション機能


###本リリース

リアクションの通知機能
運動データをグラフ表示
運動項目の連続記録を表示
連続記録の長さに合わせてバッジ
今日の気分入力
身体データの記録、共有（血圧、体重）
運動の豆知識ポップ
Health Connectで歩数を自動取得


##技術スタック
フロンエンド：Tailwind CSS：高齢者が分かりやすいUIを設定しやすい
バックエンド：Ruby
フレームワーク：Ruby on Rails：データを扱うため、CRUD操作との相性が良い
ライブラリ：devise(認証)
　　　　　　Chart.js（運動データのグラフ化）
DB：PostgreSQL：現場で多く使用されておりDBサーバーを構築・管理を学習する目的
  ：neon:コスト最適化として以降することを想定
デプロイ先：Render：無料枠で高機能なデプロイ環境を提供してくれる
環境：Docker：OSによる差をなくしセットアップを統一
